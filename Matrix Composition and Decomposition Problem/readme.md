# Matrix Expansion and Compression Problem

### Code

```cpp
#include <bits/stdc++.h>
using namespace std;

int main()
{
    int arr[100][100][100];
    int i = 0;
    for (int j = 0; j < 100; j++)
    {
        for (int k = 0; k < 100; k++)
        {
            arr[i][j][k] = (rand() % 9) + 1;
        }
    }

    int x = 99;
    for (int i = 1; i < 100; i++)
    {
        for (int j = 0; j < x; j++)
        {
            for (int k = 0; k < x; k++)
            {
                arr[i][j][k] = arr[0][j][k];
            }
        }
        x -= 1;
    }

    cout << "Please enter the Compression or Expansion number. If you want to exit give input as -1." << endl;
    cout << "PRINTNG THE MATRIX" << endl;

    int pos = 10;
    while (pos != -1)
    {
        i = 100-pos;
        for (int j = 0; j < 100; j++)
        {
            for (int k = 0; k < 100; k++)
            {
                if (arr[i][j][k] != 0)
                    cout << arr[i][j][k] << " ";
                else
                    continue;
            }
            cout << endl;
        }
        cout << endl;
        cin >> pos;
    }
}
```

### Explanation

We have created a 100 x 100 x 100 3-dimensional array in this approach. Here the zeroth index shows us a 100 x 100 array. Then index 1 would show a 99 x 99 array,  index 2 would show a 98 x 98 array and so on. It would be a stack of arrays from 100 x 100 to 1 x 1.

The first 100 x 100 array is made using a **random function generator** and the random function is **modulo by 9 + 1** so that we get values from 1 to 9. Then the first 100 x 100 array is subsequently used to form the remaining arrays.

Then the while loop gives a provision for the user to **enter the number of rows** that they want to fetch. The **square matrix** is fetched accordingly. To access the compressed or dip expanded array that number is then used in rendering the particular 2-dimensional array from the 3-dimensional array.

***NOTE :  In default first chance the matrix is compressed to 10 x 10 representation and then user inputs are considered***
